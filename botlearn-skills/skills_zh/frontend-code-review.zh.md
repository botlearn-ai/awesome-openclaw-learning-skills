## 功能说明
检查 UI 代码的一致性、布局问题和可维护性。

## 适用人群
希望保持 UI 代码一致性的前端开发者。

## 与 Playbook 的结合方式
作为学习流程中的辅助技能，融入 Playbook 的日常工作流中。

## 使用场景
你需要确保某个 UI 组件符合团队规范。

## 使用步骤
1. 提供组件或布局代码。
2. 请求 UI 最佳实践和可维护性检查。
3. 修复问题并快速重新审查。

## 示例
输入：仪表盘卡片的 React 组件。
输出：结构和样式问题的审查意见。

## 验证方式
你应该获得一份简短的 UI 改进检查清单。

## 功能说明
标记潜在问题，提出改进建议，并总结风险点。

## 适用人群
希望在 PR 审查前快速完成质量检查的工程师。

## 与 Playbook 的结合方式
作为学习流程中的辅助技能，融入 Playbook 的日常工作流中。

## 使用场景
你希望在请求团队审查前先发现潜在风险。

## 使用步骤
1. 分享你要审查的代码或 diff。
2. 询问风险、边界情况和改进建议。
3. 在 PR 审查前应用这些建议。

## 示例
输入：认证中间件的 PR diff。
输出：风险列表和修复建议。

## 验证方式
你应该获得可执行的修复建议或需要关注的风险点。
